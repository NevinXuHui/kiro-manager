# 更新日志

## 2026-06-02

- 版本号升级到 2.0.6
- 格式化导入支持仅 refreshToken、rt 字段和 `RT----social----google----邮箱` 分隔文本自动识别
- 导出文件名加入账号数量，并在导出完成后自动打开导出文件所在目录
- 自动刷新会根据检查间隔提前刷新即将过期的 Token，避免长间隔时错过刷新窗口

## 2026-05-11

- 版本号升级到 2.0.5
- 修复 Release 安装包开通 Overages 时可能丢失或误用 profileArn，导致服务端返回 unexpected error 的问题
- 本地活跃账号导入、切换账号、手动刷新和自动刷新都会保留 profileArn，避免超额开通资料丢失
- 开通 Overages 时只允许当前激活账号读取本机 Kiro profileArn，避免 accessToken 与其它账号 profileArn 错配
- 缺少 profileArn 时给出明确提示，方便重新从本机 Kiro 登录缓存导入

## 2026-05-11

- 版本号升级到 2.0.4
- 新增 Kiro Desktop Auth refreshToken 验活 fallback，缺少 clientId/clientSecret 的账号也可刷新、检查和格式化导入验活
- 格式化导入新增桌面验活提示，修复 refreshToken-only 账号被误标为仅导入或失败的问题
- 新增 Overages 超额开通能力，支持 Pro 及以上账号单个/批量开通，并在已开通时保持 $ 按钮常亮
- 优化封禁账号导入展示，保留邮箱信息并使用红色流光边框强调异常状态
- 调整账号卡片字体、字号和整体视觉层级，使用微软雅黑提升可读性，降低背景配色疲劳
- 优化刷新、检查、自动刷新链路，减少缺少 clientId/clientSecret 时的错误拦截

## 2026-05-03

- 版本号升级到 2.0.3
- 修复 Release 版本健康检查把封禁号误判为正常的问题，后端重新纳入 ListAvailableModels 封禁检测
- 优化账号验证请求链路，复用 HTTP 连接并保留 OIDC/sso-oidc 双端点 fallback

## 2026-05-03

- 版本号升级到 2.0.2
- 修复 Release 安装版默认窗口过小的问题，默认窗口调整为 1280x800，最小尺寸调整为 960x640，并允许用户自由缩放窗口

## 2026-05-02

- 版本号升级到 2.0.1
- 取消批量导入/检查/刷新的数量上限
- 更新封号检测
- 批量导入自适应多种JSON格式（数组/包装对象/扁平导出/简化凭证）
- 修复刷新账号时缺少 authMethod/provider 参数导致失败的问题
- 后端OIDC Token刷新增加双端点fallback（oidc + sso-oidc）
- UI升级为可切换的 iOS 玻璃风格双主题，优化浅色/深色模式的卡片、侧边栏、工具栏和动效
- 侧边栏底部新增日/月图标主题切换按钮，并放置在当前账号信息上方左侧
- 放大账号详情和可用模型窗口，优化模型列表多列展示和详情信息布局
- 批量导入、批量检查、批量刷新新增底部悬浮进度提示
- 修复批量健康检查异常计数与列表状态不同步的问题，并补充“已封禁”状态筛选
